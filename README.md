系统需维护以下全局配置状态：
- 目标编码器策略：默认值为 auto。
- VAAPI 设备路径：默认值为 auto。

脚本需支持接收不限数量的输入文件，并支持以下可选命令行参数：

- -q：覆盖视频压缩率设定。
- -p：激活退出时等待标识。
- -h：输出命令的帮助文档并安全退出。

针对 VAAPI 设备路径为 auto 的情况，系统需实现一个动态硬件探测机制：
1. 遍历 Linux 内核的 /sys/class/drm/renderD* 节点。
2. 顺着 sysfs 的符号链接，读取该节点在 device/driver/module 下绑定的真实驱动模块名称。
3. 匹配算法：当且仅当驱动模块名称为 amdgpu, i915, xe 或 radeon 时，认定该节点为目标设备。
4. 将首个匹配成功的节点映射为其在 /dev/dri/ 下的绝对路径，并交还给主进程。

2. 信号陷阱：拦截用户的中断信号 (SIGINT, SIGTERM)。当触发中断时，系统必须检查“当前正在处理的输出文件路径”。若该文件存在于磁盘上，必须将其彻底删除后才能让脚本退出，以防止产生损坏的视频文件。

2. 阶段二：NVENC。
   - 触发条件：当前文件未完成，且策略为 auto 或 nvenc。

3. 阶段三：CPU。
   - 触发条件：当前文件未完成，且策略为 auto 或 cpu。
