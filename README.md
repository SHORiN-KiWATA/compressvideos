# 第一部分：总体系统架构与核心特性
本系统高度依赖系统预装的 `ffmpeg` 和底层显卡驱动接口，运行环境为类 Unix 系统（重点兼容 Linux 硬件接口规范）。
系统需实现以下核心特性：
1. **纯命令行交互**：支持灵活参数解析及多文件批量处理。
2. **动态环境侦测**：自动扫描底层硬件节点与编码库能力。
3. **自动降级容错机制 (Auto-Fallback)**：在处理每个视频时，维护一个状态机，按 `Nvidia -> 通用核显(VAAPI) -> CPU软编` 的顺位进行“尝试-失败-回退”的平滑降级调度。
4. **极简画质控制与映射**：将复杂的底层质量参数（CRF/QP）抽象为 1~10 的直观数字挡位。
5. **智能媒体分流**：基于输出文件后缀自动决策是执行视频压缩还是纯音频提取。
6. **优雅的异常处理与脏数据防护**：监听系统中断信号，随时清理不完整的半成品文件。
7. 多线程并发。压缩失败的视频应该在运行结束后基于提示。

# 第二部分：模块化详细设计与底层实现规范

## 模块 1：命令行接口 (CLI) 与默认配置
系统启动时默认状态为：编码格式=h264，设备=auto，画质=5。
参数解析器需支持以下标志位，并严格区分“带参赋值”与“无参查询”两种行为：
- `-h, --help`：打印帮助文档。
- `-i, --input <file>`：输入文件，支持多次声明。缺失需报错。如果用户不适用-i选项，仅传入文件路径，自动视为输入文件。
- `-o, --output <file>`：输出文件。若未提供，基于输入名生成 `*_compressed.mp4`；若无扩展名，自动补全 `.mp4`。
- `-c, --codec [value]`：指定目标编码格式（如h264, hevc）。**无参触发行为**：仅打印当前系统/设备支持的所有编码格式列表后退出。
- `-d, --accel-device [value]`：枚举值：auto, nvidia, amdgpu, intel, cpu。**无参触发行为**：扫描底层硬件节点，列出当前机器可用计算设备后退出。
- `-q, --quality <1-10>`：限制在1-10整数范围。
- `-p, --pause`：标记位，执行完毕退出前暂停等待按键。

## 模块 2：硬件探测与挂载映射引擎
必须通过读取 Linux 底层文件节点来侦测物理设备：
- **VAAPI 通用设备探测**：
  - 扫描目录 `/sys/class/drm/renderD*`。
  - 解析路径下 `device/driver/module` 的符号链接（Symlink）真实名称。
  - 驱动名映射：`amdgpu` / `radeon` 归类为 AMD；`i915` / `xe` 归类为 Intel。
  - 挂载路径生成：提取节点编号，生成供底层引擎挂载的绝对路径 `/dev/dri/renderD[X]`。
- **Nvidia 专属探测**：
  - 探测系统 `nvidia-smi` 命令的可用性，或直接扫描是否存在 `/dev/nvidia*` 字符设备节点。

## 模块 3：编码器兼容性扫描引擎
通过调用底层引擎的查询接口，并通过标准输出 (stdout) 的文本模式匹配提取能力清单：
- **基准查询调用**：`ffmpeg -hide_banner -encoders`
- **解析提取特征**：
  - **NVENC 体系**：匹配包含 `nvenc` 且以 ` V`（代表视频）开头的行，剥离 `_nvenc` 后缀得到标准外部名称。
  - **VAAPI 体系**：匹配包含 `vaapi` 且以 ` V` 开头的行，剥离 `_vaapi` 后缀。
  - **CPU 软编体系**：探测查询结果中是否显式包含 `libx264`, `libx265`, `libvpx-vp9`, `libsvtav1`, `libaom-av1` 核心库。

## 模块 4：核心处理管道与 FFmpeg 参数拼装规范
在实际执行音视频处理任务时，所有 FFmpeg 调用必须包含全局参数 `-y`（静默覆盖）与标准输入重定向 `</dev/null`。

### 4.1 通用控制层约束
- **音频流处理**：统一绑定参数 `-c:a aac -b:a 128k`。
- **画质标尺映射算法**：将外层 1~10 映射为底层的质量绝对值。
  - 映射规则：1对应38，10对应11，步长为3（例：5->26, 6->23）。数字越小底层数值越大（高压缩率/低画质）。
- **奇数分辨率截断滤镜**：规避硬件编码器奇数尺寸报错。
  - 软编与NVENC专属参数：`-vf "scale=trunc(iw/2)*2:trunc(ih/2)*2"`
  - VAAPI 专属参数：`-vf "scale_vaapi=w=trunc(iw/2)*2:h=trunc(ih/2)*2"`

### 4.2 智能媒体分流 (Audio-only 管道)
- **触发条件**：输出路径后缀名忽略大小写后匹配 `mp3|wav|m4a|flac|aac|ogg|wma|opus`。
- **构建参数**：仅执行流分离与转换，丢弃所有视频滤镜，附加 `-vn` 参数。完成即跳至下一文件。

### 4.3 分支 A：Nvidia 硬件加速调度 (NVENC)
- **准入条件**：用户指定 auto/nvidia + 探测到 Nvidia 节点 + 兼容性扫描支持目标编码。
- **硬件初始化参数**：`-hwaccel cuda`
- **视频编码器名称构建**：`<codec>_nvenc`
- **质量与色彩控制参数**：`-preset p4 -cq <映射后的质量值> -pix_fmt yuv420p`

### 4.4 分支 B：通用显卡加速调度 (VAAPI)
- **准入条件**：上一步失败/跳过 + 用户指定 auto/amdgpu/intel + 探测到对应 /dev/dri/ 节点 + 兼容性扫描支持。
- **硬件初始化参数**：需要完整的三段式挂载：`-hwaccel vaapi -hwaccel_device <探测到的具体节点路径> -hwaccel_output_format vaapi`
- **视频编码器名称构建**：`<codec>_vaapi`
- **质量与色彩控制参数**：`-qp <映射后的质量值>`

### 4.5 分支 C：CPU 软件回退调度
- **准入条件**：硬件加速全军覆没 + 用户指定 auto/cpu + 兼容性扫描包含对应软编库。
- **视频编码器名称构建 (内部映射)**：`h264`->`libx264`, `hevc`->`libx265`, `vp9`->`libvpx-vp9`, `av1`->(首选`libsvtav1`，备选`libaom-av1`)。
- **质量与色彩控制参数**：`-crf <映射后的质量值> -preset fast -pix_fmt yuv420p`

## 模块 5：系统级进程保护与生命周期
- **环境依赖阻断**：前置检查环境变量中是否存在 `ffmpeg`，缺失则立刻引发致命错误。
- **彩色日志输出**：建立标准日志器，区分 INFO（蓝色/普通流转）、WARN（黄色/降级触发或局部失败）、ERROR（红色/阻断错误）、OK（绿色/单文件成功）。
- **信号捕获与防脏数据机制**：
  - 注册系统级别的 `SIGINT` (Ctrl+C) 和 `SIGTERM` 监听。
  - 触发回调时，精准读取当前上下文正在写入的「输出文件绝对路径」。
  - 执文件系统的物理删除操作 (`rm`/`unlink`) 销毁半成品。
  - 最后以异常状态码 `130` 结束整个进程生命周期。
